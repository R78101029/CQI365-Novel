## 格式與風格 (Do's and Don'ts)

### DO (要)

- 使用具體的細節描寫（參考 `_world/` 設定）
- **新元素首次登場時，必須進行詳細描寫**：包含外觀、聲音、質感、角色的主觀感受
- 描寫角色的內心衝突與成長

### DON'T (不要)

- 不要讓角色說出與其身份不符的語言
- 不要讓角色輕易突破世界規則的限制
- 不要出現「神奇的救援」，所有轉折都必須有伏筆

## 工作流程提醒

### 一致性檢查 (Continuity Check) - **CRITICAL**

- **交叉比對 (Cross-Reference)**：在撰寫新章節前，**必須**重新閱讀相關聯的舊章節
- **避免矛盾 (Avoid Contradictions)**：
  - 時間戳記是否一致？
  - 關鍵事件的觸發順序是否一致？
  - 物理狀態（天氣、光線、環境描述）是否一致？

**每次完成章節後，務必執行以下更新：**

1. **更新 `_meta/creative_plan.md`**：標記完成的章節為 ✅
2. **更新 `_meta/chapter_order.md`** (新增或修改章節後)
3. **更新 `_characters/timelines.md`** (若有角色狀態變化)
4. **更新 `_world/` 相關文檔** (若引入新設定)
