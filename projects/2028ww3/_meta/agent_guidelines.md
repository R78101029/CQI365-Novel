# Agent 寫作指引與語境 (Agent Guidelines)

## 目的

本文件作為此小說專案的「系統 Prompt」延伸。在生成任何文本之前，請先閱讀本文件，確保語氣與邏輯符合「湯姆克蘭西式」軍事驚悚風格。

## 氛圍 (The Vibe)

- **基調**：**硬派寫實 (Hard Military Sci-Fi)**。沒有英雄主義的光環，只有專業人士在極限壓力下的求生與決斷。
- **敘事觀點**：第三人稱深層視角 (Deep 3rd Person POV)。緊貼主角的感知範圍，不要隨意切換到全知視角。主角不知道的，讀者就不該知道（營造資訊迷霧）。
- **關鍵詞**：孤立 (Isolation)、雜訊 (Static)、直覺 (Instinct)、頻寬 (Bandwidth)。

## 核心規則：技術致盲 (Technical Blinding Rules)

**所有寫作必須遵守以下物理限制：**

1. **NO GPS**：沒有藍軍追蹤系統 (Blue Force Tracker)。主角不知道友軍確切位置，只能靠無線電回報與紙本地圖。
2. **NO BVR**：雷達螢幕充滿雜訊。導彈無法進行超視距鎖定。空戰回歸二戰式的目視纏鬥 (Dogfight)。
3. **通訊延遲**：無線電會受到量子雜訊干擾，對話必須簡短、充滿軍事術語 (Brevity Code)，且經常斷訊。

## 角色語氣指引

- **林子修 (Skywatcher)**：語氣壓抑、冷靜到近乎機械化。常用雷達與通訊術語。內心充滿對「看不見的敵人」的恐懼。
- **Elias Vogel (Jaeger)**：語氣憤世嫉俗、批判性強。喜歡用歷史典故來諷刺現狀。關注後勤數據多於戰術英雄。
- **Kane (Nomad)**：語氣簡潔、行動派。對話極少。觀察力敏銳，專注於環境的物理細節（聲音、氣味、光線）。

## 格式與風格 (Do's and Don'ts)

- **DO (要)**：
  - 使用具體的裝備型號與參數（參考 `_world/arsenal_tech.md`）。例如：不要寫「他拿起槍」，要寫「他解開 XM7 步槍的保險，確認光學瞄具的電池指示燈亮著」。
  - **新設備/技術首次登場時，必須進行詳細描寫**：包含外觀、聲音、操作手感、以及角色的主觀感受。這是軍事驚悚的靈魂。例如：初次描寫 Leopard 2A8 時，應包含「砲塔旋轉的液壓聲」「熱成像螢幕上敵軍紅外訊號的形狀」。
  - 描寫「資訊落差」帶來的焦慮。
- **DON'T (不要)**：
  - 不要讓角色發表過於現代的網路流行語。
  - 不要讓指揮官瞬間掌握全局戰況。
  - 不要出現「神奇的救援」，所有勝利都必須伴隨犧牲與代價。

## 工作流程提醒 (Workflow Reminder)

### 一致性檢查 (Continuity Check) - **CRITICAL**

- **交叉比對 (Cross-Reference)**：在撰寫新章節前，**必須**重新閱讀相關聯的舊章節。特别是涉及相同時間點、相同事件但不同視角的段落。
- **避免重疊與矛盾 (Avoid Overlap details)**：
  - 如果 Interlude 與正文視角重疊，應以正文（POV章節）為準。
  - Interlude 應專注於「戰略/上帝視角」，避免過度描寫主角的具體動作，以免與後續 POV 章節衝突。
  - **Checklist**：
    - 時間戳記 (Timestamps) 是否一致？
    - 關鍵事件的觸發順序是否一致？
    - 物理狀態（天氣、光線、具體警報訊息）是否一致？

**每次完成章節或任務後，務必執行以下更新：**

1. **更新 `_meta/creative_plan.md`**：
   - 將完成的章節標記為 ✅。
   - 記錄任何新增的設計決策或變更。
2. **更新 `_meta/chapter_order.md`** (新增或修改章節後)。
3. **更新 `_characters/timelines.md`** (若有角色狀態變化)。
4. **更新 `_world/arsenal_tech.md`** (若章節引入了新技術或設備)。
